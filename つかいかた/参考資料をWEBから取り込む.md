# 参考資料をWEBから取り込む（Obsidian Web Clipper の使い方）

> **ひとことで**：障がい特性・対処法・制度解説などの**参考記事を、ブラウザ拡張で `raw/` に集め**、そこから必要な要点を `wiki/` や `リファレンス/` に**自分の言葉で書き起こす**（蒸留する）流れです。

支援に役立つ情報はWEB上にたくさんあります。それらを集めて、自分専用の参考知識ベースに育てられます。鍵は **「集めた記事＝原本は raw に置く」「使う知識＝要約は自分で書く」** の区別です。

---

## 1. Obsidian Web Clipper とは

Obsidian 公式の**ブラウザ拡張機能**です。見ているWebページを**ワンクリックで Markdown に変換して、Vault に保存**できます。

- 対応ブラウザ：Chrome / Edge / Firefox / Safari / Brave など
- 入手：各ブラウザの拡張ストアで「**Obsidian Web Clipper**」を検索（公式）／または [clipper.obsidian.md](https://obsidian.md/clipper)
- 公式ガイド：[help.obsidian.md](https://help.obsidian.md/web-clipper)（操作の細部はバージョンで変わるので、最新は公式を参照）

## 2. 初期設定（このVault向け・おすすめ）

拡張を入れたら、最初に次を設定します。

1. **保存先のVault**を、このテンプレート（`oya-inai-wiki-template`）にする
2. **保存先フォルダ**を **`raw/WEB資料/`** にする
   - 理由：`raw/` は「原本の書庫」。集めた記事は、まず**原本として** raw に置きます。
   - `raw/` の中身は `.gitignore` で**共有・公開されません**（後述の著作権の観点でも重要）。
3. **プロパティ（frontmatter）**に、出典が後で分かる項目を自動付与すると便利：
   - `source`（元記事のURL）／`clipped`（取り込んだ日付）／`title`（記事タイトル）
4. **クリップの形式**を選ぶ：
   - **全文**：記事まるごと（読み込み重視）
   - **選択範囲／ハイライト**：必要な部分だけ（軽い・おすすめ）

> 💡 サイトごとに保存先やテンプレートを変える「テンプレート（trigger）」機能もあります。最初は既定の1つで十分です。

## 3. 集め方（WEB検索 → クリップ）

1. 検索エンジンで参考記事を探す（例：「自閉症 感覚過敏 対処」「強度行動障害 支援 事例」）
2. 良い記事を開く → ブラウザの **Web Clipper アイコン**をクリック
3. 保存先が `raw/WEB資料/` になっているのを確認 → **保存**

**集めると役立つもの（例）**
- 障がい特性の解説（感覚過敏・コミュニケーション特性 など）
- 対処法・支援技法（構造化・視覚支援・SST など）
- 制度・サービスの解説、医療情報
- → いずれも **一般的・公開の参考資料**

> ⚠️ **本人の個人情報・機微情報は、ここで扱うものではありません。** それらはWEB記事ではなく、`raw/` 直下や該当本人の記録（person_id付き）として、[[プライバシーの扱い方]] に従って扱います。

## 4. ⚠️ 著作権と共有の注意（重要）

クリップした記事は、**他の人が書いた著作物**です。あなたのVaultのコンテンツ（CC BY-SA 4.0）とは別物で、あなたが自由に再配布できるものではありません。だから：

- **記事の全文は `raw/WEB資料/` に留めます**（`.gitignore` で共有・公開されない設定）。
- **公開リポジトリや配布物・他者への共有物に、記事の全文を入れない。**
- 共有してよいのは、**あなた自身が書いた要約・考察と、出典（URL）リンク**だけです。

> これは「自分用に集めて読む」のはOK、「他人に配るのは自分の要約だけ」という整理です。

## 5. raw に取り込んだ後の流れ（蒸留）

`raw/WEB資料/` は「読むための書庫」。そこから **必要な知識だけを、自分の言葉で `wiki/` や `リファレンス/` に書き起こします**。全文コピペではありません。

### 一般的な知識（特定の本人に依らない）

→ **Concept ページ**を作る（`templates/concept.md` を使い、`wiki/concepts/` に保存）。

- 例：複数の記事から「感覚過敏への対処」を要約 → `wiki/concepts/C_感覚過敏.md`
- 本文に**自分の言葉で要点**を書き、末尾の出典に**元記事のURL**を貼る（raw の原本にもリンク）
- 同梱の `リファレンス/`（汎用22ページ）と並ぶ、**あなた自身の参考知識**になります

### 特定の本人に効きそうな対処法

→ その本人の **Protocol / Trial** に落とし込む（**person_id 付き**）。

- 例：記事で知った「予定を絵カードで予告する」を、実際に試して `T_..._P_002` に記録

### AIを使う場合

- 「`raw/WEB資料/◯◯.md` を読んで、concept ページにまとめて」と依頼。複数記事の**横断要約**も頼めます。
- 一般知識（機微情報でない）なのでクラウドAIに渡しやすいですが、**長い全文は「要約して」と指示**すると効率的です（→ [[AIと一緒に使う場合]]）。

### AIを使わない場合

- 記事を読み、要点を Concept ページの欄に書く。詳細は raw の原本に残るので、そちらを参照すればOK。
- コツ：**読みながらハイライト → ハイライトだけクリップ**しておくと、要約がぐっと楽になります。

## 6. 集めた参考資料を整理する

- `raw/WEB資料/` が増えたら、テーマ別のサブフォルダに（例：`raw/WEB資料/感覚特性/`）。
- `wiki/concepts/` に作ったページ同士を `[[ ]]` でつなぐと、**自分専用の参考知識マップ**になります。
- 同梱の `リファレンス/` ＋ 自分で集めた Concept で、現場の知識ベースが少しずつ育ちます。

## 7. 流れのまとめ

```
WEB検索
  → Web Clipper で raw/WEB資料/ に保存（原本・全文・非公開）
  → 読んで要点を wiki/concepts（または リファレンス）に蒸留
     ＝ 自分の言葉の要約 ＋ 出典リンク（全文コピペしない）
  → 本人に効くものは その本人の Protocol / Trial へ（person_id付き）
```

**全文は raw に留め、外に出すのは自分の要約だけ** — これさえ守れば、安心して参考資料を集められます。
