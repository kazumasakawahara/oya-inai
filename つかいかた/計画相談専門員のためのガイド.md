# 計画相談支援専門員のためのガイド（複数の本人を一つのVaultで管理する）

> **ひとことで**：担当する**複数の本人を、この一つのVaultでまとめて管理**できます。鍵は「本人ごとにIDをふる」こと、それだけです。

このガイドは、複数の方の計画を担当する立場の方向けです。基本の操作は [[はじめに]] と [[本人を追加する]] にあります。ここでは「複数人をどう整理するか」に絞って説明します。

## 1. 基本：本人ごとに「ID」をふる

- 担当する本人に、**P_001・P_002・P_003 …** と番号（person_id）をふります。
- 各本人の **Person ページ**が、その人の「ホーム」になります。
- すべてのページ（Trigger / Trial / Protocol / Ecomap）の先頭にある **`person_id:`** に、その本人のIDを必ず書きます。
  → これで一覧表（データベース）が**本人ごとに正しく整理**されます。**ここだけ守れば、あとは混ざりません。**

## 2. 新しい本人を追加する

いちばん簡単な方法：**記入例をコピーして書き換える**。

1. `記入例/persons/P_000_見本_あおい` を複製
2. ファイル名を `P_002_仮名` に変更（本名は使わない）
3. 先頭の `person_id:` を `P_002` に変更
4. 中身をその本人の内容に書き換える

> **本名はファイル名にも本文にも書きません。** 仮名＋ID（P_002）で扱い、本名や連絡先は `raw/` に置きます（→ [[プライバシーの扱い方]]）。

## 3. ファイル名の付け方（本人が混ざらないように）

ファイル名の**末尾に `_P_XXX`** を付けると、誰の記録か一目で分かり、別の本人と混ざりません。

| 種類 | 例 |
|---|---|
| 人物 | `P_002_仮名.md` |
| トライアル | `T_2026-05-12_入浴拒否対応_P_002.md` |
| トリガー | `TG_joy_青いタオル_P_002.md` |
| プロトコル | `PR_bath_P_002.md` |
| エコマップ | `EM_current_P_002_2026-06.md` |

## 4. 特定の本人の情報にたどり着く（3つの方法）

1. **Person ページを開く** — その人の Trigger・Trial・Protocol へのリンクが全部そこに集まっています。
2. **一覧表（データベース）で絞り込む** — `データベース/0_データベースについて` を開き、表の見出し（「本人」列など）を**クリックして並べ替え・絞り込み**。担当者・本人別に見られます。
3. **検索する** — 検索窓で `P_002` と打てば、その本人のページだけ集まります。

> グラフビューでも、本人ごとに記録がまとまって見えます。

## 5. 日々の使い方（アセスメント・計画作成に効く）

- 面談やモニタリングでの気づきは、その場で**該当本人の Trial / Trigger** に書きます。
- サービス等利用計画の各欄に、この記録がそのまま対応します（→ [[purpose]] §2-1）：
  - 「興味・関心・強み」→ Person の人格の核・Joy Trigger
  - 「合理的配慮」→ Protocol・Distress Trigger
  - 「これまでの経過」→ Trial（**失敗の記録**が特に効く）
  - 「本人の意向」→ 意思決定支援の素材（→ [[CD_意思決定支援]]）
- **「要確認」の一覧表**で、「しばらく更新していない本人」「古くなったプロトコル」を把握できます。担当が多いほど効きます。

## 6. フォルダの整理（2通り。どちらでもOK）

| 方式 | 内容 | 向き |
|---|---|---|
| **既定（型ごと）** | `wiki/persons/` に全員の人物ページ、`wiki/trials/` に全員のトライアル…。person_id と一覧表で本人別に絞る | 少人数〜中規模。迷ったらこれ |
| **本人ごと** | `wiki/persons/P_002_仮名/` のように、その人の記録を一か所にまとめる | 担当が多く、人別に開きたいとき |

**どちらでも、一覧表とリンクは `person_id` で動きます。** 大事なのはフォルダの形ではなく、**person_id を必ず付けること**です。

## 7. リファレンスは全員で共有

`リファレンス/`（制度・概念）は**全本人で共通**です。本人ごとに作り直す必要はありません。地域の窓口情報だけ、ご自分の地域版に書き換えてください（→ `記入例/地域の例（北九州市）`）。

## 8. プライバシー（複数の本人を扱うからこそ）

- `raw/` には複数の本人の原本が混ざります。**特に厳重に**（非公開・[[プライバシーの扱い方]]）。
- 記録を誰かに渡すときは、**その本人の分だけ**を、本人（または成年後見人）の同意の範囲で。
- AIに手伝わせる場合も、本人を取り違えないよう、person_id を明示して依頼します。

## AIと一緒に使う場合

面談メモを渡して「**P_002 さんの Trial にして**」のように、**person_id を指定**して頼めば、正しい本人の記録として整えてくれます（→ [[AIと一緒に使う場合]]・機微情報の注意も同ページ）。

---

*迷ったら原則はひとつ：**本人ごとに person_id をふり、各ページに必ず書く**。それだけで、何人担当しても整理は崩れません。*
